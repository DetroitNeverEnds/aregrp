# Общие правила проекта AREGRP

## Архитектура проекта

- Монорепозиторий с frontend (React + TypeScript) и backend (Django)
- Frontend использует Vite, React Router, SASS
- Backend использует Django Ninja для API

## Стандарты кодирования

### Frontend

- **Язык**: TypeScript (строгий режим)
- **Стили**: SASS/SCSS с модульным подходом
- **Компоненты**: Функциональные компоненты с хуками
- **Роутинг**: React Router v6+
- **Интернационализация**: react-i18next

### Стили (SCSS)

- Использовать переменные из `frontend/src/styles/variables.scss`
- Минимизировать дублирование через:
  - Вынос общих значений в переменные
  - Наследование классов через `@extend`
  - Миксины для повторяющихся паттернов
- Именование классов: BEM методология
- Один файл стилей на компонент
- используем последние стандарты (например, @use)
- используй модули (module.scss)

### Структура компонентов

```
ComponentName/
├── ComponentName.tsx       # Основной компонент
├── ComponentName.scss      # Стили компонента
├── ComponentName.test.tsx  # Тесты
├── ComponentName.stories.tsx # Storybook истории
├── index.ts               # Экспорты
```

### Импорты

- Использовать именованные импорты
- Группировать импорты: библиотеки → компоненты → стили → типы
- Использовать алиасы путей из tsconfig

### Тестирование

- Покрытие тестами всех компонентов
- Использовать Vitest + Testing Library
- Тесты должны быть читаемыми и понятными

### Документация

- Не надо писать readme если я сам не попрошу
- JSDoc комментарии для публичных API
- Storybook истории для визуальной документации
- при написании сторибука использовать специальные импорты '@storybook/react-vite'
- Все примеры пишутся только для сторибука, не надо писать их в проекте

## Готовые компоненты

Перед созданием нового компонента проверить наличие готовых компонентов.
Надо избегать написания новых стилей, если хочется написать новый, надо уточнить, нужно ли это делоать

## Git

- Коммиты на русском языке
- Формат: `тип: краткое описание`
- Типы: feat, fix, docs, style, refactor, test, chore

## Дополнительно:

- не надо запускать сторибук или дев сервер, они уже запущены
- не надо писать документацию для задачи, если не попросили
- стандартный отступ - 4 для всех файлов
- перед созданием нового компонента согласуй список пропсов
